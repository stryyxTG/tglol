# TGLol Bot

Telegram-бот на `aiogram` + `Telethon` для хранения Telegram-аккаунтов, распределения аккаунтов по воркерам и выдачи прокси.

Используй бот только для аккаунтов, к которым у тебя есть законный доступ.

## Возможности

- Добавление аккаунтов по номеру телефона и коду входа.
- Добавление аккаунтов через `.session`.
- Добавление аккаунтов через `.session + .json`.
- Массовое добавление аккаунтов через `.zip`.
- Умное сопоставление `.session` и `.json` внутри ZIP.
- Сохранение оригинального JSON без обрезания полей.
- Автогенерация desktop JSON, если JSON не был загружен.
- Общее хранилище аккаунтов с разделами `НЕРЕГ` и `РЕГ`.
- Хранилище каждого воркера с разделами `НЕРЕГ` и `РЕГ`.
- Массовая выдача аккаунтов воркеру из общего хранилища.
- Возврат аккаунтов от воркера обратно в общее хранилище.
- Скачивание аккаунтов из общего хранилища ZIP-архивом.
- Удаление аккаунтов по одному или массово.
- Перехват кодов из `Verification Codes` для воркера и владельца.
- Добавление и выдача прокси воркерам.
- Поддержка нескольких владельцев через `OWNER_IDS`.

## Требования

- Python 3.11+
- Telegram Bot Token от `@BotFather`
- `TELEGRAM_API_ID` и `TELEGRAM_API_HASH` с https://my.telegram.org

## Установка

```bash
git clone https://github.com/stryyxTG/tglol.git
cd tglol

python3 -m venv venv
source venv/bin/activate

pip install -r requirements.txt
```

Для Windows:

```powershell
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
```

## Настройка `.env`

Создай файл `.env` в корне проекта:

```env
BOT_TOKEN=PUT_BOT_TOKEN_HERE
OWNER_IDS=123456789,987654321

TELEGRAM_API_ID=123456
TELEGRAM_API_HASH=put_api_hash_here

BOT_PARSE_MODE=HTML
DATA_DIR=storage
```

Можно использовать `ADMIN_IDS`, но лучше указывать владельцев через `OWNER_IDS`.

Несколько владельцев указываются через запятую, пробел или точку с запятой:

```env
OWNER_IDS=111111111,222222222,333333333
```

## Запуск

```bash
python bot.py
```

Запуск на сервере через `nohup`:


Проверить лог:

```bash
tail -f bot.log
```

## Как пользоваться

1. Напиши боту `/start` с аккаунта владельца.
2. Открой раздел `Аккаунты`.
3. Выбери, куда добавлять аккаунты:
   - `Основное хранилище`
   - конкретному воркеру
4. Все новые аккаунты сначала попадают в `НЕРЕГ`.
5. В карточке аккаунта можно перенести его в `РЕГ` или обратно в `НЕРЕГ`.

## Добавление аккаунтов

Доступные способы:

- `По номеру + код`
- `.session`
- `.session + .json`
- `Массово .zip`

При добавлении через ZIP архив может быть плоским, без папок:

```text
79586161128.session
79586161128.json
79586163636.session
79586163636.json
```

Если JSON есть, бот сохраняет его оригинальным и при выдаче отдаёт именно его.
Если JSON нет, бот генерирует desktop JSON на Windows/Linux профиле.

## Воркеры

Владелец может:

- добавить воркера по Telegram ID;
- указать имя воркера;
- выдать воркеру аккаунты;
- смотреть `НЕРЕГ` и `РЕГ` воркера;
- удалять аккаунты воркера;
- массово вернуть аккаунты воркера в общее хранилище.

Воркер может:

- видеть только свои аккаунты;
- получать код из `Verification Codes`;
- переносить аккаунт между `НЕРЕГ` и `РЕГ`;
- получать выданные ему прокси.

## Прокси

Формат прокси:

```text
185.189.245.51:17240:user335792:9etya2
185.189.245.67:17240:user335792:9etya2
```

Каждая строка — отдельный прокси.

Логика:

1. Владелец добавляет один или несколько прокси.
2. Прокси появляются в общем списке владельца.
3. Владелец нажимает `Выдать воркеру`.
4. Выбирает воркера и количество.
5. Прокси пропадают из общего списка.
6. Воркер открывает `Прокси` и получает их по одному.

После каждой новой выдачи счётчик прокси для воркера считается заново.

## Хранилище файлов

По умолчанию данные лежат в папке `storage`:

```text
storage/
  bot.sqlite3
  sessions/
  json/
  tmp/
```

Что где лежит:

- `bot.sqlite3` — база данных.
- `sessions/` — `.session` файлы аккаунтов.
- `json/` — оригинальные или сгенерированные JSON.
- `tmp/` — временные файлы загрузки.

При удалении аккаунта из бота строка удаляется из базы, а файлы аккаунта удаляются с сервера. Telegram-сессия в самом Telegram при этом не разлогинивается.
